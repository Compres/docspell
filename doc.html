Documentation

Docspell can organize large amounts of PDF files that are typically scanned paper documents. You can associate tags, set correspondends, the “topic”, a name, a date etc.. If your PDFs are associated with this meta data, you should be able to quickly find a document later using the search feature. But adding this manually to each document is a tedious task. What if most of it could be attached automatically?

How it works

Documents have two main properties: a correspondent (sender or receiver that is not you) and something the document is about. Usually it is about a person or some thing – maybe your car, or contracts concerning some familiy member, etc.

  1. You maintain a kind of address book. It should list all possible correspondents and the concerning people/things. Of course, this usually grows incrementally.
  2. When docspell analyzes the document content, it tries to find matches within your address book. It can detect the correspondent and a concerning person or thing. It will then associate this data to your documents.
  3. You can then inspect what docspell has done and correct it. If docspell has found multiple suggestions, they will be shown for you to select one. If it is not correctly associated, very often the correct one is just one click away.

The set of meta data that docspell uses to draw suggestions from, has still be maintained manually. But usually, this data doesn’t grow the same way as the documents. After a while there is a quite complete address book and only once in a while it has to be revisited to add or change something.

Terms

In order to simplify these pages, some terms should be explained first.

An Item is roughly your (pdf) document, only that an item may consist of multiple files, called attachments. And an item has meta data associated:

  • a correspondent: the other side of the communication. It can be an organization or a person.
  • a concerning person or equipment: a person or thing that this item is about. Maybe it is an insurance contract about your car.
  • tag: an item can be tagged with custom tags. A tag can have a category. This is intended for grouping tags, for example a category doctype could be used to group tags like bill, contract, receipt etc. Usually an item is not tagged with more than one of such tags.
  • a item date: this is the date of the document – if this is not set, the created date of the item is used.
  • a direction: one of “incoming” or “outgoing”
  • a name: some item name, defaults to the file name of the attachments
  • some notes: arbitraty descriptive text. You can use markdown here, which is appropriately formatted in the web application.

Besides items and their meta data, the users of the application are part of a collective. The idea is, that very often more than one user needs access to those items. A collective is a group of users that share items. For instance, a family could be a collective and its members are the users. Thus the account name is twofold: it consists of a collective name and a user name.

Limitations

  • Docspell currently supports only PDF files.
  • The PDF view relies soley on the browsers capabilities. Sadly, not all browsers can display PDF files. Some may require extra plugins. And it’s especially sad, that mobile browsers wont’t display the files. It works with the major desktop browsers (firefox, chromium), though.

Similiar Software

There are many DMS – document management systems, that provide much much more features, but are also more complex to use and setup. Docspell is not a DMS.

One project that is very close is paperless, written in Python, that also uses tesseract/unpaper.
